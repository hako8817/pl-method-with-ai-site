# 한장 요약 v1

이 문서는 `00.AI 제안서 자동화 방법론` 전체를 한 번에 보는 요약본이다.  
프로세스는 `제안 요청서 분석 -> 요구사항 대응 구조화 -> 제안 전략/목차 설계 -> 본문 초안/완성 -> 검토/제출 -> 착수 입력 전환`으로 두고, 각 단계에서 `중요 task / AI가 할 것 / 사람이 할 것`을 함께 본다.

## 1. 제안 요청서 분석
- 중요 task
  - 사업 맥락 파악
  - 요구사항 추출
  - 제출 형식/평가 항목 파악
  - 질문/위험 신호 정리
- AI가 할 것
  - 문서 분해
  - 요구사항 분류
  - 누락 질문 후보 생성
- 사람이 할 것
  - 고객 의도 해석
  - 수주 가능성/위험 판단

## 2. 요구사항 대응 구조화
- 중요 task
  - 요구사항 클러스터링
  - 대응 메트릭스 작성
  - 확정/질문/유보 항목 분리
- AI가 할 것
  - 중복/누락 탐지
  - 메트릭스 초안 작성
  - 본문 연결 포인트 제안
- 사람이 할 것
  - 대응 수준 결정
  - 약속 범위 결정

## 3. 제안 전략과 목차 설계
- 중요 task
  - 제안 전략 수립
  - 차별화 포인트 정리
  - 목차와 섹션 흐름 설계
- AI가 할 것
  - 목차 초안 생성
  - 메시지 구조 초안 제안
- 사람이 할 것
  - 강조 포인트 선택
  - 평가 기준과 발표 흐름 정렬

## 4. 본문 초안과 완성
- 중요 task
  - 섹션별 초안 작성
  - 근거/사례 배치
  - 문체와 논리 정합성 보정
- AI가 할 것
  - 섹션 초안 생성
  - 표/목록 정리
  - 반복 문장 탐지
- 사람이 할 것
  - 과장 표현 제거
  - 실제 수행 가능성 검토
  - 고객 맥락 보강

## 5. 검토와 제출 직전 체크
- 중요 task
  - 요구사항 누락 점검
  - 목차/본문/전략 모순 점검
  - 범위/책임/리스크 최종 확인
- AI가 할 것
  - 누락/모순 탐지
  - 반복 표현/약한 메시지 탐지
- 사람이 할 것
  - 최종 승인
  - 범위/일정/비용 함의 판단

## 6. 제안 발표 준비와 실행 (11장)
- 중요 task
  - 발표 전략 수립 (핵심 메시지 3개 이내 + 심사위원 특성 파악)
  - 발표 구조 설계 (첫 1~2분 핵심 메시지 선언 → 대응 → 차별화 → 클로징)
  - 예상 질문 준비 (10장 위험 구간 → Q&A 질문으로 전환)
  - 리허설 (시간 체크 + 핵심 메시지 전달 여부 확인)
- AI가 할 것
  - 발표 구조 초안 생성 (섹션별 시간 배분 + 오프닝·클로징 멘트 초안)
  - 예상 질문 목록 생성 (유형별 분류 + 답변 핵심 포인트 초안)
- 사람이 할 것
  - 발표 전략 최종 결정 (핵심 메시지, 강조 포인트 선택)
  - 현장에서 답하기 어려운 질문 분리 및 서면 후속 제출 원칙 공유
  - Q&A 담당자 배분 (기술/사업/일정 구분)

## 7. 제안 이후 착수 입력 전환 (12장)
- 중요 task
  - 제안서 해석
  - 착수 질문 리스트 정리
  - RIB 입력 후보 전환
- AI가 할 것
  - 범위 문장 추출
  - 킥오프 질문 초안 생성
  - RIB 입력 후보 정리
- 사람이 할 것
  - 실제 약속 해석
  - 범위 확정/위험 공개 판단

## 7. 이 책의 핵심 원칙
- 제안서는 수주형 프로젝트의 첫 기획 산출물이다.
- 기획은 순서대로 수행하지만, 설계는 원하는 아웃풋에서 거꾸로 시작한다.
- AI는 구조화와 초안 생성을 가속하고, 사람은 약속과 책임을 판단한다.
- 좋은 제안서는 제출에서 끝나지 않고, 착수와 이후 기획으로 이어진다.
