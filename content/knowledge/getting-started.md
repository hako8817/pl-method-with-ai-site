# README_사용순서

## 공개 메모
- 이 문서는 Knowledge를 처음 볼 때의 시작 가이드다.
- 내부 운영 규칙 문서와 자동화 프로세스 도식은 공개 대상에서 제외한다.

## 1. 시작 순서
1. [Knowledge](index)
2. 현재 공개된 시작 문서 확인

## 2. Knowledge 작업 시작점
- Daily나 프로젝트에서 새 워딩이 나오면 먼저 `기존 노트 검색`부터 한다.
- 그 다음 `신규 생성 / 기존 업데이트 / 보류`를 결정한다.
- 결정 후에만 노트 작성으로 들어간다.

## 3. 현재 기준
- 기존 노트를 먼저 검색한다.
- 신규 생성 / 기존 업데이트 / 보류를 구분한다.
- 관련 노트와 연결 가능성을 함께 본다.

## 4. 작업 완료 기준
- 노트 생성 또는 업데이트가 실제로 끝났는가
- 관련 링크와 대상 파일이 확인됐는가
- Daily 또는 MOC 반영이 필요한 경우 같이 끝냈는가
