# PL method with ai

기획자의 전체 프로세스를 `제안 -> 기획 -> PKM -> Knowledge` 관점으로 정리한 공개 허브다.

이 사이트는 제안서 작성, 기획 방법론, PKM 구조, Knowledge 샘플을 연결해서 기획자의 산출물 흐름을 보여준다.

## 읽기 시작
1. [전체 프로세스](process/)
2. [제안서 작성](proposal/)
3. [기획 방법론](planning/)
4. [PKM](pkm/)
5. [Knowledge](knowledge/)

## 섹션

### 전체 프로세스
제안, 기획, PKM, Knowledge를 한 흐름으로 묶어 보는 탐색 허브.

[전체 프로세스](process/)

### 제안서 작성
수주형 프로젝트의 앞단에서 `RFP 분석 -> 제안 전략 -> 제안서 작성`을 다루는 방법론.

[제안서 작성](proposal/)

### 기획 방법론
산출물 체인을 기준으로 기획 문서와 업무 흐름을 다시 설계하는 방법론.

[기획 방법론](planning/)

### PKM
개인지식관리를 메모 습관이 아니라 운영 체계로 보는 방법론.

[PKM](pkm/)

### Knowledge
기획자의 비즈니스기획, 디자인, 기술 지식을 구조화한 지식 베이스.

[Knowledge](knowledge/)

## 공개 범위
- 현재 v1은 `process`, `proposal`, `planning`, `pkm`, `knowledge` 중심으로 연다.
- `Decision OS`는 폴더 단위 비공개다.
